BEKCAN API Dokümantasyonu

Uygulamaya Dön

Genel Bakış

Bu dokümantasyon, BEKCAN Çoklu Konferans (Multi-Tenant) Yönetim Sistemi API'sinin nasıl kullanılacağını açıklar. API iki ana kategoriye ayrılır:

  • Public API: Herkese açık verileri çeker (Konferans takvimi, konuşmacılar vb.). Kimlik doğrulama gerektirmez.
  • Admin API: Konferans verilerini yönetmek (CRUD) için kullanılır. JWT (JSON Web Token) kimlik doğrulaması gerektirir.

Kimlik Doğrulama (Admin)

Admin endpoint'lerine istek yaparken, Authorization header'ı içerisinde Bearer <TOKEN> göndermeniz gerekmektedir.

POST /api/auth/login

Kullanıcı adı ve şifre ile JWT token alır.

Örnek İstek (cURL):

curl -X POST https://conf.bekcan.com/api/auth/login \
-H "Content-Type: application/json" \
-d '{"username": "admin", "password": "admin123"}'

Başarılı Yanıt:

{
  "token": "eyJhbGciOiJIUzI1NiIsInR...",
  "role": "superadmin",
  "conference_id": null
}

Public API (Açık Erişim)

GET /api/conferences/:slug/schedule

Belirtilen kısa ada (slug) sahip konferansın tüm oturum ve konuşmacı verilerini getirir. Mobil uygulama (Frontend) ana ekranda bu veriyi kullanır.

Örnek İstek:

curl -X GET https://conf.bekcan.com/api/conferences/bekcan2026/schedule

POST /api/conferences/:slug/attendee

Cihaz kimliği ile yeni katılımcı kaydı oluşturur.

curl -X POST https://conf.bekcan.com/api/conferences/bekcan2026/attendee \
-H "Content-Type: application/json" \
-d '{
  "device_id": "uuid-v4-string",
  "full_name": "Can Bekcan",
  "email": "info@bekcan.com"
}'

Admin API - Veri Yönetimi

Not: Organizatör rolündeki kullanıcılar sadece kendilerine atanmış conference_id için işlem yapabilirler. Aşağıdaki örneklerde :id konferans kimliğidir.

POST /api/admin/conferences/:id/speakers

Konferansa yeni konuşmacı ekler.

curl -X POST https://conf.bekcan.com/api/admin/conferences/1/speakers \
-H "Authorization: Bearer <TOKEN>" \
-H "Content-Type: application/json" \
-d '{
  "speaker_ref": "sp1",
  "full_name": "Dr. Ahmet Yılmaz",
  "title": "Doçent",
  "institution": "Örnek Üniversitesi"
}'

POST /api/admin/conferences/:id/sessions

Konferansa yeni oturum ekler ve konuşmacıları bağlar.

curl -X POST https://conf.bekcan.com/api/admin/conferences/1/sessions \
-H "Authorization: Bearer <TOKEN>" \
-H "Content-Type: application/json" \
-d '{
  "session_ref": "s1",
  "title_tr": "Yapay Zeka ve Gelecek",
  "title_en": "AI and the Future",
  "start_time": "2026-06-18T09:00:00Z",
  "end_time": "2026-06-18T10:30:00Z",
  "room": "Salon A",
  "category": "Keynote",
  "speaker_ids": ["sp1"]
}'

DELETE /api/admin/speakers/:id

Mevcut bir konuşmacıyı (ve bağlı olduğu session ilişkilerini) siler.

curl -X DELETE https://conf.bekcan.com/api/admin/speakers/5 \
-H "Authorization: Bearer <TOKEN>"

POST /api/admin/conferences/:id/import-json

Mevcut bir schedule.json dosyasını doğrudan veritabanına aktarır (Mevcut verileri ezer!).

curl -X POST https://conf.bekcan.com/api/admin/conferences/1/import-json \
-H "Authorization: Bearer <TOKEN>" \
-H "Content-Type: application/json" \
-d @schedule.json
